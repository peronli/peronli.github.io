FreeRTOS 基础知识

不加入 OS （操作系统）的单片机编程通常称为裸机编程

RTOS (Real-time Operating System 实时操作系统) 是一种体积小巧、确定性强的计算机操作系统, 通常用于需要在严格时间限制内对外部事件做出反应的嵌入式系统。

常用的 RTOS 有国外的FreeRTOS、 μC/OS、RTX、Huawei LiteOS 和 AliOS-Things 等。

多任务系统

在裸机系统中，程序的主体是 CPU 按照顺序执行的。而在多任务系统中， 任务的执行是由系统调度的。

任务

任务是一个独立的函数，函数主体无限循环且不能返回。

在多任务系统中，每个任务都是独立的，互不干扰的，所以要为每个任务都分配独立的栈空间，这个栈空间通常是一个预先定义好的全局数组， 也可以是动态分配的一段内存空间，但它们都存在于 RAM 中。在多任务系统中，有多少个任务就需要定义多少个任务栈。

任务栈：

任务栈是一个预先定义好的全局数据，数据类型为StackType_t，大小由 TASK1_STACK_SIZE 这个宏来定义， 默认为 128，单位为字，即 512字节，这也是 FreeRTOS 推荐的最小的任务栈。

任务状态：

  • 运行态（runnnig）：当任务正在运行，CPU的使用权被这个任务占用
  • 就绪态（ready）：任务已经具备了运行条件（没有被挂起或阻塞），但是又更高优先级或同优先级的任务正在运行，所以需要等待的状态。
  • 阻塞态（blocked）：任务在等待信号量、消息队列、事件标准组、系统延时时，被称为阻塞态，如果等待的事件到了，就会自动退出阻塞态，准备运行
  • 挂起态（suspended）：任务被暂时停止，通过调用挂起函数（vTaskSuspend()）可把指定任务挂起，任务挂起后暂时不会运行，只有调用恢复函数（xTaskResume()）才可以退出挂起状态

队列

队列是为了任务与任务、任务与中断之间的通信而准备的，可以在任务与任务、任务与中断之间传递消息。

写队列和读队列都是采用复制的方式，队列的操作都是采用先进先出的的方式（FIFO）。

FreeRTOS 中的信号量的是依据队列实现的。

信号量

信号量用于同步，任务间或者任务和中断间同步。

信号量本质是只有计数值的队列，当计数上限为1时称为二值信号量（Binary Semaphores），否则为计数信号量（Counting Semaphores）。

在 CubeMX 中，需在 FreeRTOS--Config parameter 中使能 USE_COUNTING_SEMAPHORES 才能创建计数信号量。

互斥量用于互锁，用于保护同时只能有一个任务访问的资源，为资源上一把锁。

FreeRTOS 编程风格

FreeRTOS 数据类型

在 FreeRTOS 中， 使用的数据类型虽然都是标准 C 里面的数据类型，但是针对不同的处理器，对标准 C 的数据类型又进行了重定义。

FreeRTOS 中详细的数据类型重定义在 portmacro.h 这个头文件中实现

FreeRTOS 变量名的定义

在 FreeRTOS 中，定义变量的时候往往会把变量的类型当作前缀加在变量上， 这样的好处是让用户一看到这个变量就知道该变量的类型。

比如 char 型变量的前缀是 c， short 型变量的前缀是 s， long 型变量的前缀是 l， portBASE_TYPE 类型变量的前缀是 x。

无符号型变量的前缀为 u， 、指针变量的前缀为 p。

因此，当我们定义一个无符号的 char 型变量的时候会加一个 uc 前缀， 当定义一个 char 型的指针变量的时候会有一个 pc 前缀

还有其他的数据类型，比如数据结构，任务句柄，队列句柄等定义的变量名的前缀也是 x。

FreeRTOS 函数名的定义

函数名包含了函数返回值的类型、 函数所在的文件名和函数的功能，如果是私有的函数则会加一个 prv 的前缀。示例：

  1. vTaskPrioritySet()：返回值为 void 型， 在task.c 这个文件中定义。
  2. xQueueReceive()：返回值为 portBASE_TYPE 型， 在 queue.c 这个文件中定义。
  3. vSemaphoreCreateBinary()：返回值为 void 型， 在 semphr.h 这个文件中定义。

FreeRTOS 宏定义

通用宏定义如下：

  • pdTRUE 1
  • pdFALSE 0
  • pdPASS 1
  • pdFAIL 0

一般宏定义由大写字母表示，并配有小写字母的前缀， 前缀用于表示该宏在哪个头文件定义。

STM32CubeMX 实战

三板斧

展开左侧 System Core： - SYS：Debug 选择 Serial Wire；时基选除了Systick以外的即可，以防与FreeRTOS重合 - RCC：HES 选择外部晶振

打开时钟树，配置 HCLK（MHZ）为最大值（注：下方有提示最大值是多少），然后点两下空格即自动配置完成。

FreeRTOS 配置

展开左侧 Middleware and Software Packs，点击 FreeRTOS：

上方的 Mode ：Interface 选择 CMSIS_V1 或 CMSIS_V2

下方的 Configuration：

点击 Tasks and Queues：

新增任务，这时会让我们编辑该任务的所有属性： - Task Name：任务名 - Priority：优先级：数字越大优先执行 - Stack Size：执行任务所需空间大小 - Entry Function：入口函数 - Code Generation Option：代码生成：一般用默认，weak为弱定义可被覆盖 - Parameter：入口函数形参，一般没有填 NULL - Allocation：动态或静态内存创建，动态即空间大小会自动分配，静态需自己指定。初学者选动态即可，选静态则可填以下两个选项： - Buffer Name：缓冲区名称 - Control Block Name：控制块名称

新增队列,这时会让我们编辑该任务的所有属性： - Queue Name：队列名 - Queue Size：队列长度 - Item Size：队列每个数据大小 - Allocation：动态或静态内存创建，动态即空间大小会自动分配，静态需自己指定。初学者选动态即可，选静态则可填以下两个选项： - Buffer Name：缓冲区名称 - Control Block Name：控制块名称
